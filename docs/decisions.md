# decisions.md

## פורמט הלוגים (כפי שממומש כרגע)
- אני מתייחס ללוג כקובץ טקסט שמחולק לשורות.
- "שגיאה" נספרת אם המחרוזת ERROR מופיעה בשורה (לא משנה איפה).
- "אזהרה" נספרת אם המחרוזת WARN מופיעה בשורה (לא משנה איפה).
- הספירה היא case-insensitive (מומר ל-Uppercase לפני בדיקה).
- בכל שורה נספרת לכל היותר 1 ERROR ו-1 WARN (גם אם מופיע כמה פעמים באותה שורה).

## Assumptions
- אין פרסור רשמי לטיימסטמפ/רמות/מבנה JSON; זו רק ספירת מילות מפתח.
- קבצים יכולים להכיל תווים לא תקינים ל-UTF-8, ולכן קוראים עם errors="replace".
- זה MVP לצרכי תרגול: דיוק “לוג-פורמטים אמיתיים” יגיע בשלב הבא.

## השלב הבא
- להגדיר פורמט לוג רשמי (regex / JSON / syslog) ולחלץ שדות (timestamp, level, module).
- להוסיף אפשרות סינון/פלט JSON ב-CLI (למשל `--json`).
- להרחיב בדיקות לעוד פורמטים וקבצים גדולים (streaming במקום read_text).
